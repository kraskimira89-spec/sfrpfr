# ТЗ: B2C-сервис SFRFR (частные клиенты)

**Версия:** 0.1  
**Дата:** 2026-07-22  
**Статус:** согласовано к реализации MVP  
**Связанные документы:** [b2c-monetization-model.md](b2c-monetization-model.md), [b2c-customer-journey.md](b2c-customer-journey.md), [b2c-architecture.md](b2c-architecture.md), [contracts/](contracts/)

## 1. Цель

Создать легальный B2C-сервис для частных клиентов: диагностика пенсионного дела, подготовка документов и сопровождение до результата. Клиент самостоятельно подаёт заявление в СФР / МФЦ / Госуслуги. Монетизация — гибридная: фикс + постоплата за подтверждённый результат.

## 2. Термины

| Термин | Определение |
|--------|-------------|
| Клиент | Физлицо: пенсионер, инвалид, родственник или законный представитель |
| Дело (`case`) | Карточка пенсионного кейса в SFRFR |
| Диагностика | Первичный разбор документов и план действий |
| Сопровождение | Подготовка пакета, чек-лист, консультации, контроль сроков |
| Success fee | Постоплата только при подтверждённом повышении пенсии / ЕДВ |
| Первое обращение | Дата фиксации первого обращения клиента в систему |
| CRM | [«Клиентская база» Taganay](https://taganay.clientbase.ru/login.php) |
| Sheets | Обезличенная управленческая таблица Google Sheets |

## 3. Сегменты клиентов

1. Пенсионеры и инвалиды — проверка стажа и перерасчёт.
2. Родственники / представители — сбор документов и контроль подачи.
3. Самозанятый консультант работает с ними напрямую (B2B/НКО — вне scope MVP).

## 4. Ценность

- Выявление неучтённого стажа и ошибок в данных.
- Индивидуальный чек-лист и черновики заявлений.
- Сопровождение до результата без ложных гарантий.
- Прозрачная оплата: фикс заранее, success fee — только при подтверждённом результате.

## 5. Правовые ограничения (обязательные)

1. Сервис **не гарантирует** перерасчёт и размер выплат.
2. Сервис **не подаёт** заявления от имени клиента и не имитирует органы СФР.
3. Клиент подаёт документы **самостоятельно** через СФР / МФЦ / Госуслуги.
4. На сайте — публичная оферта; на каждое дело — индивидуальное соглашение-заказ.
5. Success fee начисляется **только** при документально подтверждённом повышении пенсии и/или ЕДВ.
6. Постоплата — через **2–3 месяца** после перерасчёта и фактического получения повышенной пенсии.
7. Если через **6 месяцев** после первого обращения клиент не сообщает результат — уведомления и подготовка пакета для взыскания (после юридической проверки формулировок).
8. До публикации оферты — проверка юристом (ЗоЗПП, 152-ФЗ, допустимость success fee).

## 6. Состав услуги MVP

### 6.1. Диагностика

- Приём заявки и согласие на ПДн.
- Загрузка документов (ИЛС, трудовая, справки, МСЭ и др.).
- OCR + первичный аудит.
- План действий и индивидуальный чек-лист.
- Фиксированная оплата.

### 6.2. Сопровождение

- Подготовка черновиков заявлений / запросов.
- Инструкция самостоятельной подачи.
- Контроль сроков и напоминания.
- Фиксация результата и расчёт success fee.
- Фиксированная оплата + постоплата по результату.

### Вне MVP

- Подача представителем по доверенности.
- B2B / НКО / white-label.
- Автоматическая интеграция API СФР (официального публичного API нет).

## 7. Пользовательские сценарии (кратко)

См. полный путь: [b2c-customer-journey.md](b2c-customer-journey.md).

1. Лендинг / MAX → заявка → согласие → лид в CRM.
2. Оплата диагностики → загрузка документов → аудит → чек-лист.
3. Акцепт оферты + индивидуальный заказ → оплата сопровождения.
4. Выдача пакета → клиент подаёт сам.
5. Загрузка решения → подтверждение эксперта → счёт success fee (через 2–3 мес.).
6. Оплата / закрытие; при молчании 6 мес. — эскалация.

## 8. Функциональные требования

| ID | Требование |
|----|------------|
| F1 | Публичная заявка без полного доступа к ПДн в логах |
| F2 | Согласие на обработку ПДн до загрузки документов |
| F3 | Создание дела и синхронизация с CRM Taganay по `case_id` |
| F4 | Индивидуальный чек-лист дела с задачами, сроками, статусами |
| F5 | Загрузка документов в защищённое хранилище |
| F6 | Статусы дела + финансовые статусы |
| F7 | Оплата диагностики и сопровождения (платёжный провайдер + чек) |
| F8 | Выдача пакета: черновики + инструкция подачи |
| F9 | Фиксация результата и расчёт success fee |
| F10 | Журнал уведомлений клиенту (в т.ч. 6-месячный срок) |
| F11 | Обезличенная выгрузка в Google Sheets |
| F12 | Публикация оферты; генерация индивидуального соглашения-заказа |

## 9. Нефункциональные требования

- ПДн: шифрование at rest где применимо, маскирование в логах, RLS.
- Доступ: клиент — только свои дела; эксперт — назначенные; service role — только сервер.
- Аудит доступа к документам.
- Срок хранения документов — по политике (по умолчанию из `.env`: `DATA_RETENTION_DAYS`).
- Google Sheets **не** содержит ПДн и ссылок на файлы.

## 10. CRM, чек-листы, Sheets

| Система | Роль | Что хранит |
|---------|------|------------|
| SFRFR | Источник истины по делу и документам | ПДн, файлы, чек-лист, расчёты |
| Taganay CRM | Воронка, этапы, задачи, ответственные | Контакты, стадии, `case_id` |
| Google Sheets | Управленческая витрина | Только обезличенные поля |

Поля Sheets (разрешено): `case_id`, сегмент, регион укрупнённо, этап, тип проблемы, даты, тариф, факт оплаты, диапазон результата, агрегаты.  
Запрещено: ФИО, телефон, адрес, СНИЛС, паспорт, сканы, тексты документов, URL файлов.

## 11. Критерии приёмки MVP

1. Описаны оферта и шаблон индивидуального соглашения с формулой постоплаты.
2. Заявка → CRM → дело → чек-лист работает end-to-end на стенде.
3. Оплата фикса проходит с выдачей чека (или статусом фискализации).
4. Success fee считается только после подтверждения результата экспертом.
5. Счёт success fee выставляется не ранее чем через 2 месяца после подтверждённого повышения.
6. При отсутствии ответа 6 месяцев от первого обращения — запись в журнале и пакет для эскалации.
7. Sheets получает только whitelist-поля без ПДн.
8. Клиент не может «подать в СФР» через сервис от своего имени автоматически.

## 12. Этапы реализации

1. ТЗ и тарифная модель (этот комплект документов).
2. Схема Supabase + RLS + Storage.
3. Интеграция CRM Taganay.
4. Заявка, согласие, дело, чек-лист.
5. Обезличенная выгрузка в Sheets.
6. Платежи и фискализация.
7. OCR/аудит → пакет клиенту.
8. Результат и success fee.
9. Тесты и пилот.
